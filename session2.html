2 - Exercice : fetching sequences from command line

  1. Open a terminal; type fetch-sequences. The result should be an error message indicating that no genome information has been provided. However, this means that the RSAT tools are installed and can be accessed through the terminal.
  2. Examine the options of the fetch-sequence tool by running fetch-sequences -h
    3. The long list of options is typical for all RSAT command-line tools. This is the sign that these tools can be tuned using many parameters, but also implies that they are more complex to use. Many of these options have default values, meaning that predefined values will be used if no information for these options are provided by the user (optional parameters). However, some parameters have no default values and need to be provided by the user (as the genome information previously)
  3. Try to identify the parameters which are compulsory.
  4. Run the fetch-sequence tool using the bed file for the dog genome peaks.
You should now have a file in your working directory containing the fasta sequences for the dog peaks.

Choosing the right background model

Goal: The choice of a background model is critical for the quality of the results. The goal of this part are:
  • learn how to generate a background model based on Markov models
  • test the effect of using different background models
  • experiment with negative controls

1 - create background models

Generating a realistic background model is a tricky task! Ideally, the background model should be realistic i.e. capture possible features from the input datasets (e.g. compositional biases such as CpG content, etc...), without being "too close" to the dataset, as the signal we are looking for would be included in the background model. Unrealistic background models are for example random sequences based on equiprobable nucleotides. On the other hand, random sequences based on Markov models appear to be a good trade-off, and are fast to compute.
The RSAT distribution contains pre-computed background models for various Markov model orders
